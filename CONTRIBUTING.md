## 📝 Стандарты кода

### Backend (C# / ASP.NET)

```csharp
// ✅ Правильно
public class ChatService
{
    private readonly ILogger<ChatService> _logger;

    public ChatService(ILogger<ChatService> logger)
    {
        _logger = logger;
    }

    public async Task<Message> SendMessageAsync(Guid chatId, string text)
    {
        // Используйте var для локальных переменных
        var message = new Message
        {
            Id = Guid.NewGuid(),
            ChatId = chatId,
            Text = text
        };

        _logger.LogInformation("Sending message to chat {ChatId}", chatId);

        return message;
    }
}

// ❌ Неправильно
public class chatService  // должно быть PascalCase
{
    public async Task<Message> sendmessageAsync(...)  // должно быть PascalCase
    {
        var Message = new Message();  // переменные в camelCase
    }
}
```

**Правила**:
- PascalCase для классов, методов, свойств
- camelCase для переменных, параметров
- Префикс `I` для интерфейсов
- Суффикс `Async` для асинхронных методов
- XML-документация для публичных API

### Frontend (TypeScript / React)

```typescript
// ✅ Правильно
interface MessageProps {

export const MessageCard: React.FC<MessageProps> = ({ id, text, isMine }) => {

  id: string;

const messageCard = (props) => {  // компонент должен быть PascalCase
  const [isOpen, setisOpen] = useState(false);  // setIsOpen
  return <div>{props.text}</div>;  // используйте деструктуризацию
```

**Правила**:
- PascalCase для компонентов, интерфейсов, типов
- camelCase для переменных, функций, свойств
- kebab-case для имён файлов компонентов
- Используйте TypeScript строго (no implicit any)
- Следуйте правилам ESLint

### Python (Telegram Bot)

```python
# ✅ Правильно
class ChatService:
    def __init__(self, session: AsyncSession):
        self.session = session

    async def get_chat(self, chat_id: int) -> Optional[Chat]:
        """Получить чат по ID."""
        result = await self.session.get(Chat, chat_id)
        return result

# ❌ Неправильно
class chatService:  # должно быть PascalCase
    def __init__(self, session):  # добавьте типизацию
        self.session = session

    async def GetChat(self, chat_id):  # должно быть snake_case
        pass
```

**Правила**:
- PascalCase для классов
- snake_case для функций, переменных, методов
- UPPER_CASE для констант
- Type hints для всех функций
- Docstrings для публичных API

## 📑 Структура коммитов

### Формат коммита

```
<type>(<scope>): <subject>

<body>

<footer>
```

### Типы коммитов

- `feat` — новая функция
- `fix` — исправление ошибки
- `docs` — документация
- `style` — форматирование, отступы
- `refactor` — рефакторинг без изменений функциональности
- `test` — добавление тестов
- `chore` — сборка, зависимости, инструменты

### Примеры

```bash
# Новая функция
git commit -m "feat(notifications): добавить звуковое сопровождение уведомлений"

# Исправление ошибки
git commit -m "fix(backend): исправить подсчёт непрочитанных сообщений"

# Документация
git commit -m "docs(readme): обновить инструкцию по установке Ollama"

# Рефакторинг
git commit -m "refactor(frontend): упростить логику обработки SignalR"
```

### Подробный коммит

```bash
git commit -m "feat(notifications): добавить системные уведомления

- Интеграция Web Notification API
- Обновление счётчика через SignalR
- Звуковой сигнал при новом сообщении
- Иконки и badge для уведомлений

Closes #42"
```
